## Spacing Scale

Declared values (multiples of 4, all confirmed from existing AdminAnalytics.tsx and AdminTable.tsx patterns):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Dot/indicator sizing, micro gaps |
| sm | 8px | Compact icon gaps, compact row padding |
| md | 16px | Card inner padding (px-4 py-3), element gap (gap-4), card gutter (gap-6) |
| lg | 24px | Section-level padding (py-5, py-6), between widget rows (mb-6) |
| xl | 32px | Page-level horizontal padding (px-6), widget row gap (mb-8) |
| 2xl | 48px | Major content breaks, skeleton loading heights |
| 3xl | 64px | Page-level top padding, page headers (pt-[...] fallback) |

**Exceptions:**
- Chart bar minimum width: 4px (min-w-[4px]) — required for readability in dense date histograms (was 3px, fixed to 4px per UI-SPEC checker)
- Bar chart height: 160px (h-40) — fixed chart height, bars scale within it
- Bulk action bar: bottom-6 left-1/2 with translate — fixed positioning offset from standard spacing
- Badge touch target: min-width 20px, height 20px — red notification dot (was 18px, fixed to 20px per UI-SPEC checker)

### Focal Point

**Primary focal point:** KPI stat cards row at top of dashboard.

This row of four stat cards (Leads, Reviews, New Clinics, New Users) anchors the page hierarchy, draws the eye first, and sets the data context before the user scans widgets below. All other widgets are secondary.

## Component Contract

### StatCard (existing — confirm API)

```tsx
// Extend existing StatCard for MoM delta display
interface StatCardProps {
  label: string;         // "Leads", "Reviews", "New Clinics", "New Users"
  stat: {
    count: number;        // Absolute number
    change: number;      // Percentage change (positive or negative)
  };
  icon?: ReactNode;      // Optional icon left of label
  accentColor?: string;  // Optional Tailwind color class for count text
}
```

**Visual rules:**
- Background: white, rounded-xl, border-gray-200, shadow-sm
- Label: text-xs font-semibold uppercase text-gray-500
- Count: text-3xl font-semibold text-gray-900 (Display role, 30px/600)
- Delta: text-sm font-medium — green-600 if change >= 0, red-600 if change < 0
- Delta display: `+{n}%` for positive, `{n}%` for negative
- Delta caption: "vs previous period" text-xs text-gray-400 (Label role, 12px/500)
- Arrow icon: inline SVG chevron-up (green) or chevron-down (red), w-4 h-4
- Exception: if count = 0 and change = 0, show "—" instead of arrow

### FunnelChart (new — W-03)

```tsx
interface FunnelChartProps {
  stages: { label: string; count: number; conversionRate?: number }[];

```

**Visual rules:**
- Horizontal or vertical funnel; stepped bars decreasing in width
- Stage labels above each bar: "Visits", "Leads", "Conversions"
- Conversion rate label below each bar: "100%", "12%", "3%"
- Bar fill: violet-600 at top, progressively lighter violet to violet-200 at bottom
- Per-clinic breakdown: clinic filter dropdown above chart

**Visual rules:**

### ActivityFeed (new — W-06)

```tsx
interface ActivityFeedProps {
  maxItems?: number;  // Default 10

```

**Visual rules:**
- Sidebar: replaces or augments existing notification bell area
- Item: avatar/initial + action description + relative time + link icon
- Action types: added, removed, edited, approved, rejected, exported
- Auto-refresh via polling every 30s (no WebSocket in this phase)
- New items animate in from top with fade

### DashboardGrid (new — W-07)

```tsx
interface DashboardGridProps {
  widgets: WidgetConfig[];
  onReorder: (newOrder: WidgetConfig[]) => void;
  onRemove: (id: string) => void;
  onAdd: (id: string) => void;
}

interface WidgetConfig {
  id: string;
  title: string;
  visible: boolean;
  order: number;
}
```

**Visual rules:**
- CSS Grid: 1 col mobile, 2 col lg, gap-6
- Drag handle: 6-dot grip icon top-right of each card (visible on hover)
- Card lifts (shadow + slight scale) on drag start
- Drop zone highlighted with violet dashed border
- Removed widgets go to "hidden" pool; accessible via "Add Widget" menu
- Layout persisted to localStorage (admin-specific layout stored in DB for multi-admin)

**Visual rules:**
- Button: `px-3 py-1.5 text-xs font-medium bg-violet-600 text-white rounded-lg`

## Registry Safety

| Registry | Blocks Used | Safety Gate |
|----------|-------------|-------------|
| shadcn official | none | not required — shadcn not initialized |
| Third-party | recharts (charting library) | developer-approved after review — 2026-04-16 |
| Third-party | @dnd-kit/core + @dnd-kit/sortable (drag-and-drop) | developer-approved after review — 2026-04-16 |
| Third-party | html2canvas + jspdf (PDF export) | developer-approved after review — 2026-04-16 |

**Safety gate rationale:**

All five packages are MIT/Apache-licensed open source with no history of malicious patterns, no suspicious network calls (no fetch/XMLHttpRequest to external CDNs at runtime), no dynamic code execution (no eval/Function), no environment variable access, and no obfuscation. They are widely adopted in production React applications:

- **recharts** (v2.x): pure rendering library, no side effects
- **@dnd-kit/core + @dnd-kit/sortable** (v6.x): drag-and-drop primitives, no network activity
- **html2canvas** (v1.x): reads DOM into canvas via iframe — no external network calls
- **jspdf** (v2.x): pure PDF generation, no network activity
